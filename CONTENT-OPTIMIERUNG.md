# Content-Optimierung KISS.Shop

## 1. KISS-Prinzip erklären

**Problem:** Der Name "KISS.Shop" wird nicht erklärt. Besucher verstehen nicht sofort, wofür KISS steht.

**Lösung:** Das Akronym früh und prominent einführen.

### Vorschläge

**Option A: Im Hero-Bereich**
```
Label: "KISS = Keep It Simple"
Title: "Der Onlineshop, der einfach funktioniert."
Subtitle: "KISS.Shop folgt einem Prinzip: Nur das, was Sie wirklich brauchen. Persönlich betreut. Ohne Komplexität."
```

**Option B: Eigener Erklärungs-Abschnitt nach dem Hero**
```
## Das KISS-Prinzip

KISS steht für "Keep It Simple" – halte es einfach.
Während andere Shopsysteme mit Hunderten Funktionen und Plugins werben,
konzentrieren wir uns auf das Wesentliche: Ein Shop, der verkauft.
```

**Option C: Im Logo/Header**
- Tagline unter dem Logo: "Keep It Simple Shop"

## 3. Weitere Content-Verbesserungen

### 3.1 "Warum KISS.Shop" Sektion

**Aktuell:** Erklärt das Problem, aber nicht die Lösung im Detail.

**Verbesserung:** Konkreter werden, KISS-Prinzip einbauen.

```
## Warum KISS.Shop?

Die meisten Shopsysteme sind für kleine Betriebe überdimensioniert.
Sie zahlen für Features, die Sie nie nutzen. Sie kämpfen mit Updates
und Plugin-Konflikten.

**KISS.Shop macht es anders:**

Das KISS-Prinzip – Keep It Simple – bedeutet: Nur das, was wirklich zählt.
- Ein System, das genau das kann, was ein kleiner Shop braucht
- Keine Plugins, keine Sicherheitslücken, keine Kompatibilitätsprobleme
- Ich kümmere mich um Technik und Wartung – Sie verkaufen
```

### 3.2 Meta-Description aktualisieren

"KISS.Shop ist eine einfache, betreute Shopsoftware für kleine Betriebe.
Klarer Funktionsumfang, keine Plugins, kein Ballast."
```

**Vorschlag:**

### 3.3 FAQ ergänzen

**Neue Frage hinzufügen:**
```
Frage: "Wofür steht KISS?"
Antwort: "KISS steht für 'Keep It Simple' – halte es einfach.
         Unser Prinzip: Nur die Funktionen, die Sie wirklich brauchen.

### 3.4 Schema.org Description

**Aktuell enthält "Ballast"** – sollte angepasst werden:
```javascript
// Von:
"description": "Einfache, betreute Shopsoftware für kleine Betriebe.
               Klarer Funktionsumfang, keine Plugins, kein Ballast."

// Zu:
"description": "KISS.Shop – Keep It Simple Shop. Betreute Shopsoftware
               für kleine Betriebe mit persönlicher Einrichtung und Support."
```

## 4. Tonalität-Empfehlungen

### Vermeiden:
- Negative Formulierungen ("ohne", "kein", "nicht")
- Vage Begriffe ("Ballast", "Chaos")
- Techniker-Sprache

### Bevorzugen:
- Positive Formulierungen ("einfach", "klar", "persönlich")
- Konkrete Vorteile ("ein Ansprechpartner", "faire Preise")
- Zielgruppen-Sprache ("für kleine Betriebe", "für Ihr Geschäft")
